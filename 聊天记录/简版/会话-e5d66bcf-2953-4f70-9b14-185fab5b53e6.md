## 工作原理

| 时机 | 动作 |
|------|------|
| 你发送消息时 | 记录用户提问，写入简版 Markdown |
| AI 回复结束时 | 追加 AI 最终回复 |
| 会话结束时 | 将完整 JSON 写入 `raw/` |
| 每次 AI 回复 / 会话结束 | 自动 `git commit` 聊天记录（已配置 upstream 时还会 push） |

与 IOM 的差异：未接入小说相关的 `afterFileEdit` 自动提交；`auto-commit` 只提交 `聊天记录/` 和 `.cursor` 配置；`.cursor/hooks/state/` 已加入 `.gitignore`，临时状态不会进仓库。

## 使用说明

1. **重启 Cursor** 或重新打开本项目，确保 Hooks 加载（也可在 **Cursor Settings → Hooks** 中确认）。
2. 正常对话即可，记录会自动出现在 `聊天记录/`。
3. 需要把配置纳入版本库时，提交 `.cursor/` 和 `聊天记录/README.md` 即可。

**说明**：Hooks 只在 Cursor Agent 对话中生效；本次对话若是在配置完成前进行的，可能不会自动写入。重启后新发一条消息，即可在 `聊天记录/简版/` 看到对应会话文件。
