## 工作原理

| 时机 | 动作 |
|------|------|
| 发送消息时 | 记录用户提问，写入 `聊天记录/简版/` |
| AI 回复结束时 | 追加 AI 最终回复，并尝试自动 commit |
| 会话结束时 | 将完整 JSON 写入 `聊天记录/raw/` |

`auto-commit.sh` 只会提交 `聊天记录/` 和 `.cursor` 相关配置；`.cursor/hooks/state/` 仍在 `.gitignore` 中，临时状态不会进仓库。

## 额外调整

- 将根目录的 `gitignore` 重命名为 `.gitignore`，使忽略规则生效（原先文件名缺少前导点，Git 不会读取）。

## 使用说明

1. **重启 Cursor** 或重新打开本项目，在 **Cursor Settings → Hooks** 中确认已加载。
2. 之后正常对话即可，记录会自动写入已有的 `聊天记录/` 目录。
3. 若需纳入版本库，可提交 `.cursor/` 与 `聊天记录/README.md`；配置好 `git push -u` 后，Hook 会在每次 AI 回复后自动 commit 并 push。

**说明**：Hooks 仅在 Cursor Agent 对话中生效；本次对话若发生在配置完成之前，可能不会自动写入。重启后发一条新消息即可验证。
